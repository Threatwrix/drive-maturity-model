Overview

In today's threat landscape, organizations face an ever-growing challenge: securing critical data and identities across increasingly complex environments. Netwrix DRIVE provides a structured, measurable path to security maturity—transforming overwhelming security requirements into clear, actionable steps.

Built on proven methodologies from ANSSI and PingCastle, DRIVE helps organizations take a regimented approach to data and identity security. Rather than attempting to fix everything at once, DRIVE prioritizes risks by their exploitability timeline—addressing immediate threats first, then building toward comprehensive, state-of-the-art protection. This ensures your most critical information stays secure while you systematically strengthen your overall security posture.

Whether you're just starting your security journey or looking to validate advanced controls, DRIVE provides the roadmap, measurements, and compliance mappings to guide your progress with confidence.

Maturity Levels

Organizations progress through 5 maturity levels based on passing ALL threshold requirements. Each check defines multiple thresholds that can block different levels.

1

Critical Exposure Eliminated

Critical Hours to Days 0-20 points

Organization has eliminated critical security exposures that pose immediate threat. All critical-severity thresholds across all checks must pass.

Example Blocking Conditions:
  • Any anonymous edit links to any content
  • Anonymous links to Critical/Highly Confidential data
  • Admin accounts without MFA
  • Clear-text password storage
  • Forest functional level < Windows Server 2008
2

High Risk Mitigated

High Weeks to Months 21-40 points

High-risk exposures exploitable within weeks are addressed. All Level 1 AND Level 2 thresholds must pass.

Example Blocking Conditions:
  • >10 anonymous links to Confidential data
  • Privileged accounts without MFA
  • Guest accounts with Edit on Sensitive data
  • Stale admin accounts (inactive >90 days)
  • Forest functional level < Windows Server 2012 R2
3

Security Baseline Established

Medium Months to Year 41-60 points

Industry baseline security practices met. All Level 1, 2, AND 3 thresholds pass.

Example Blocking Conditions:
  • >50 permanent anonymous links
  • Missing data classification on >20% of sensitive data
  • No conditional access policies for risky locations
  • Legacy authentication enabled
  • Forest functional level < Windows Server 2016
4

Enhanced Security Posture

Low Strategic Threats 61-80 points

Proactive security management with automation and advanced controls. All Level 1-4 thresholds pass.

Example Blocking Conditions:
  • Guests without access review in past 90 days
  • No automated data labeling
  • No privileged identity management (PIM)
  • Manual permission management
  • Forest functional level < Windows Server 2019
5

State-of-the-Art Security

Optimal Future-Proofed 81-100 points

State-of-the-art security with predictive capabilities. All checks at all levels pass (perfect score).

Requirements:
  • Zero anonymous links of any kind
  • Automated policy-as-code enforcement
  • Predictive threat analytics active
  • Continuous compliance validation
  • Latest security features enabled
